## Artifact System  
## The Crate Institute of Design  
## Introduction  
The Artifact System is the organizational framework used to classify, document, and reference materials throughout The Crate Institute of Design.  
Its purpose is to provide structure for collections, archives, research documents, publications, and observations.  
The system allows individual artifacts to exist within a larger body of knowledge.  
Every object becomes part of a connected archive.  
  
⸻  
  
## Core Principle  
The Institute does not collect objects.  
The Institute collects evidence.  
The Artifact System exists to preserve that evidence and make it discoverable.  
Every artifact should be traceable.  
Every document should be referenceable.  
Every observation should be connected to a larger context.  
  
⸻  
  
## Accession Numbers  
Every archived artifact receives a unique accession number.  
Format:  
CID.YYYY.XXXX  
Example:  
CID.2026.0047  
Where:  
* CID = Crate Institute of Design  
* YYYY = Year Entered Into Archive  
* XXXX = Sequential Archive Number  
The accession number remains permanently attached to the object record.  
It should never be reassigned.  
  
⸻  
  
## Collection Codes  
Artifacts are grouped into collections.  
Each collection receives a short identifier.  
Examples:  

| Code  | Collection             |
| ----- | ---------------------- |
| TYPO  | Typography             |
| INDU  | Industrial Design      |
| PACK  | Packaging              |
| SIGN  | Signage                |
| PUBL  | Publications           |
| SKATE | Skate Culture          |
| MUSIC | Record & Music Culture |
| DIGI  | Digital Products       |
| PHOTO | Photography            |
| TRANS | Transportation Systems |
  
Collection identifiers help establish context within the archive.  
  
⸻  
  
## Object References  
Individual objects may be referenced using both collection and accession identifiers.  
Example:  
MUSIC-CID.2026.0047  
or  
TYPO-CID.2026.0123  
This format immediately communicates category and archival record.  
  
⸻  
  
## Archive Status Codes  
Artifacts may be assigned preservation statuses.  

| Code | Status             |
| ---- | ------------------ |
| ACT  | Active Collection  |
| DOC  | Documented         |
| RES  | Under Research     |
| CAT  | Catalogued         |
| DIG  | Digitized          |
| REF  | Reference Material |
| RET  | Retired Archive    |
  
Status codes provide insight into the current state of documentation.  
  
⸻  
  
## Research Files  
Research projects receive independent identifiers.  
Format:  
RR-YYYY-XXX  
Example:  
RR-2026-014  
Meaning:  
Research Report Year Sequential Research Number  
Examples:  
RR-2026-014 RR-2026-027 RR-2027-003  
Research reports may reference multiple artifacts and collections.  
  
⸻  
  
## Field Notes  
Field observations are recorded using Field Note identifiers.  
Format:  
FN-YYYY-XXX  
Example:  
FN-2026-041  
Field notes may include:  
* Observations  
* Sketches  
* Photography  
* Measurements  
* Interviews  
* Environmental documentation  
Field notes serve as primary research records.  
  
⸻  
  
## Collection Studies  
Collection studies receive the designation:  
CS-YYYY-XXX  
Example:  
CS-2026-009  
A Collection Study examines relationships between multiple artifacts within a single collection.  
Examples:  
CS-2026-009 Swiss Airline Timetables  
CS-2026-010 Skateboard Graphics 1993–1998  
  
⸻  
  
## Object Cards  
Every archived object should be capable of appearing on a standardized Object Card.  
Object Cards typically contain:  
* Object Title  
* Accession Number  
* Collection  
* Date  
* Creator  
* Location  
* Material  
* Description  
* Research Notes  
* Status Code  
Object Cards act as the primary interface between artifacts and researchers.  
  
⸻  
  
## Archive Labels  
Archive labels should be concise and informational.  
Example:  
OBJECT TITLE  
MUSIC-CID.2026.0047  
Collection: Record Culture  
Status: CAT / DIG  
The purpose of labels is identification rather than promotion.  
  
⸻  
  
## Departmental Prefixes  
Institutional departments may use standardized identifiers.  
Examples:  
ARC Archive Division  
RES Research Division  
PUB Publications Division  
COL Collections Division  
VIS Visual Systems Division  
EDU Educational Resources Division  
These prefixes allow future expansion of the Institute.  
  
⸻  
  
## Publications  
Official Institute publications receive publication numbers.  
Format:  
PUB-YYYY-XXX  
Examples:  
PUB-2026-001  
The Design Annual  
PUB-2026-002  
Field Guide to Record Sleeves  
Publications become permanent records within the archive.  
  
⸻  
  
## Classification Philosophy  
The purpose of classification is not restriction.  
The purpose of classification is discovery.  
Systems should help researchers find connections between artifacts.  
The archive should remain structured enough to navigate and flexible enough to evolve.  
The classification system exists to support curiosity rather than constrain it.  
  
⸻  
  
## Closing Statement  
The Artifact System provides the organizational framework through which The Crate Institute of Design preserves knowledge.  
It transforms collections into archives.  
Observations into records.  
Documents into references.  
And individual artifacts into part of a larger and continually expanding body of understanding.  
The archive grows.  
The system remains.  
The work continues.
